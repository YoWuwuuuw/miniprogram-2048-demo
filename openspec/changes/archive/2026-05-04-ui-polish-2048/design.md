## Context

当前 2048 小程序的排行榜和"我的"页面存在以下问题：
1. 排行榜标题与表头"分数"文字轻微重叠（标题 Y 坐标偏低）
2. Top 3 排名仅用简单金/银/铜色文字，不够醒目
3. 列表项样式简陋，无卡片感、无行间距
4. 两个页面的列表风格不统一

## Goals / Non-Goals

**Goals:**
- 排行榜标题上移，与表头保持足够间距
- Top 3 使用渐变色圆角卡片背景，视觉上与其他排名拉开差距
- 列表项增加圆角卡片、行间距、微阴影，形成卡片列表效果
- "我的"页面列表项与排行榜风格统一

**Non-Goals:**
- 不改变数据结构或云端接口
- 不添加新功能（如搜索、筛选）
- 不改变滚动逻辑本身（已有实现）

## Decisions

### 1. 列表项卡片样式方案

**决定**: 使用圆角矩形背景 + 行间距 + 微阴影，替代当前的平铺条纹背景。

**理由**: 当前 `drawCloudRankItem` 和 `drawRankItem` 使用纯色填充整个行宽，无圆角无间距，视觉上像表格而非现代卡片列表。改为圆角卡片 + 间距后，每个列表项独立成"卡片"，层次感更强。

**替代方案**: 保持现有条纹但加深对比色 — 效果有限，不够现代化。

### 2. Top 3 高亮方案

**决定**: Top 3 使用渐变背景色（金 #FFD700→#FFA500、银 #C0C0C0→#A0A0A0、铜 #CD7F32→#8B5A2B），配合白色粗体文字。

**理由**: 渐变比纯色更有层次感，且与 2048 方块的暖色调风格一致。白色文字在深色渐变上对比度高，可读性好。

### 3. 标题位置调整

**决定**: 将排行榜标题 Y 坐标从 `screenHeight * 0.10` 调整为 `screenHeight * 0.06`，并适当缩小标题字号。

**理由**: 0.10 的比例在小屏设备上会让标题与表头过于接近。上移后留出足够空间。

### 4. 测试数据方案

**决定**: 在 `cloud.js` 中添加一个 `seedTestData` 函数，在排行榜进入时若数据不足 10 条则自动填充模拟数据（仅开发环境）。

**理由**: 用户数据不足时无法看到滚动和 Top 3 效果，需要测试数据验证 UI。

## Risks / Trade-offs

- [渐变背景] 不同设备上 Canvas 渐变渲染性能略有差异 → 使用简单线性渐变，避免复杂径向渐变
- [圆角卡片] 行间距会略微减少可见条目数 → 行间距控制在 4-6px，影响可接受
- [测试数据] 开发环境数据污染 → 使用明确的 mock 标识，仅在数据不足时注入
