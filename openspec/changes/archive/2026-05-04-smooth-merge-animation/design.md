## Context

当前2048游戏的合并动画实现：
- 时长：100ms
- 缓动函数：easeOutBack
- 缩放范围：1.0 -> 1.2 -> 1.0
- 实现方式：单个Tween动画，在onUpdate中计算缩放值

用户反馈动画不够丝滑，需要优化视觉效果。

## Goals / Non-Goals

**Goals:**
- 提升合并动画的流畅度和视觉效果
- 保持动画性能稳定（60fps）
- 与游戏整体风格协调

**Non-Goals:**
- 不改变游戏核心逻辑
- 不添加新的动画类型
- 不修改其他动画（移动、出现）

## Decisions

### 1. 动画时长优化
- **决定**：将合并动画时长从100ms增加到200ms
- **理由**：更长的动画时间允许更平滑的过渡，减少视觉上的"突兀感"
- **替代方案**：150ms（折中方案，但可能仍显仓促）

### 2. 缓动函数选择
- **决定**：使用自定义的弹性缓动函数，结合easeOut和轻微回弹效果
- **理由**：纯easeOutBack可能过于生硬，自定义函数可以更好地控制弹性程度
- **替代方案**：easeOutCubic（更平滑但缺乏弹性感）

### 3. 缩放范围调整
- **决定**：调整为 1.0 -> 1.15 -> 1.0，减少最大缩放幅度
- **理由**：过大的缩放（1.2）会显得突兀，1.15更柔和
- **替代方案**：保持1.2但增加动画时长（但会显得拖沓）

### 4. 多阶段动画
- **决定**：保持单阶段动画，但优化缩放曲线
- **理由**：多阶段动画会增加复杂度，且当前实现已足够表达合并效果
- **替代方案**：两阶段动画（放大+缩小分开控制）- 增加复杂度但效果提升有限

## Risks / Trade-offs

- **风险**：动画时长增加可能影响游戏节奏感
  - **缓解**：200ms仍在可接受范围内，不会明显拖慢游戏
- **风险**：缓动函数调整可能影响性能
  - **缓解**：使用简单的数学公式，避免复杂计算
