# UI/UX 优化方案

## 背景

2048 游戏核心玩法已完成，动画系统运行良好。但用户反馈以下体验问题需要优化。

## 问题分析

### 1. 远距离方块合并动画不丝滑（已完成）

**现状**: `createMoveAnimation` 使用固定 120ms 时长，无论方块移动 1 格还是 3 格，时长相同。

**目标**: 移动时长与距离成正比，远距离移动有更充裕的过渡时间。

### 2. 按钮布局不够美观，缺少功能入口（已完成）

**现状**: 仅"新游戏"和"排行榜"两个按钮，宽度占比 40% 但间距和视觉层级单一。

**目标**: 增加"设置"按钮，优化按钮尺寸和间距比例。

### 3. 弹窗样式不够精致（已完成）

**现状**: 弹窗使用纯白背景 + 简单圆角，按钮样式与游戏整体风格不够统一。

**目标**: 优化弹窗底色、阴影、按钮样式，使其与 2048 游戏的整体复古暖色风格一致。

### 4. 排行榜页面严重 bug（需修复）

**现状（截图确认）**:
- 标题 "游戏记录" 被灵动岛遮挡，只显示一个"录"字
- 用户数据卡片太窄，中文文字截断（"总局数"显示为"局"）
- 表头文字截断（"时间"显示为"时"）
- 分数使用 24px 超大字体，列表项互相重叠
- 按钮文字显示异常

**目标**: 修复所有 UI bug，使排行榜页面正常显示。

### 5. 排行榜功能拆分（新增需求）

**现状**: "排行榜"按钮进入的是用户本地历史记录页面，不是真正的排行榜。

**目标**:
- 拆分为两个按钮："排行榜"（云端多用户排行）和"我的"（本地用户数据）
- 排行榜接入微信云开发，显示所有用户的分数排名，支持滑动浏览
- "我的"页面显示用户个人统计（总局数、最高分、总步数）和历史游玩记录

### 6. 微信云开发接入（新增需求）

**现状**: 项目未接入微信云开发，所有数据仅存在本地 `wx.storage`。

**目标**:
- 初始化 `wx.cloud`
- 创建云数据库集合存储用户分数
- 游戏结束后自动上传分数到云端
- 排行榜页面从云端查询 Top 分数

## 预期效果

- 远距离合并动画与近距离一样丝滑自然
- 按钮区域更协调，新增"设置"入口
- 弹窗视觉更精致，与游戏风格统一
- 排行榜页面 UI 正常，无截断/重叠
- 排行榜显示全网用户分数排名
- "我的"页面显示个人数据和历史记录
