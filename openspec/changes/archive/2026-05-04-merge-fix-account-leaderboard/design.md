## Context

微信小游戏 2048 已有完整的游戏逻辑、动画系统、云端排行榜。当前面临四个问题：
1. 之前的合并动画修复（隐藏静止方块）导致移动和合并效果基本不可见，需要回滚
2. 账号绑定方式不合理：启动时强制弹窗影响体验，应改为用户主动登录
3. 排行榜无条目限制：数据只增不减
4. 排行榜表头随列表滚动被覆盖，小幅度滑动跳页

代码结构：`game-core.js`（纯逻辑）→ `game-scene.js`（场景+动画编排）→ `renderer.js`（Canvas 绘制）→ `cloud.js`（云数据库）。

## Goals / Non-Goals

**Goals:**
- 回滚合并动画修复，恢复原始动画效果
- 实现"我的"页面登录按钮，用户主动绑定微信
- 未登录用户分数不计入排行榜
- 排行榜限制 100 名，支持排名替换
- 排行榜固定表头 + 平滑滚动

**Non-Goals:**
- 不修改合并逻辑本身
- 不实现完整的用户资料编辑功能
- 不添加好友系统或社交功能
- 不实现动画设置选项（留作后期功能）

## Decisions

### D1: 合并动画回滚

**选择**：回滚 `game-scene.js` 中 `mergeTargets`/`_isMergeAnimating` 相关代码，恢复原始渲染逻辑。

**理由**：隐藏静止方块的方案导致移动和合并动画基本不可见，视觉效果反而变差。原始动画（滑动+脉冲+闪光）虽然有轻微重叠但整体效果更好。

**后期计划**：在设置页面中添加动画效果选项，用户可选择"默认动画"或"关闭合并动画"。

### D2: 账号绑定方式 — "我的"页面登录按钮

**选择**：在"我的"页面新增"登录"按钮，用户主动点击触发 `wx.getUserProfile`。

**流程**：
1. "我的"页面标题下方显示按钮：未登录显示"登录"，已登录显示"已登录"
2. 点击"登录"按钮调用 `wx.getUserProfile({desc: '用于排行榜昵称显示'})`
3. 成功获取后存入本地缓存 + 云端 `users` 集合
4. 失败或拒绝则使用默认昵称"匿名用户"
5. 页面底部小字说明："未登录用户的分数不会计入排行榜"

**替代方案**：启动时强制弹窗 — 体验突兀，用户可能不理解用途。

**理由**：用户主动登录更自然，在"我的"页面有明确上下文。

### D3: 未登录用户分数不计入排行榜

**选择**：在 `uploadScore()` 开头检查本地缓存的昵称，若为空或"匿名用户"则跳过上传。

**理由**：排行榜需要可识别的玩家身份，匿名用户不应占用排名位。

### D4: 排行榜 100 名限制实现

**选择**：在云端上传时进行排名判断和清理。

**实现方式**：
1. `uploadScore()` 始终上传新分数（不再仅高分时更新）
2. 上传前查询当前第 100 名的分数
3. 若新分数 ≤ 第 100 名分数且已有 ≥ 100 条记录，则不上传
4. 上传后，若该用户已有旧记录则删除旧记录
5. 定期清理排名 > 100 的记录（在上传成功后触发）

**数据模型变更**：`scores` 集合新增字段：
- `nickname`: string — 玩家昵称
- `_openid`: string — 已有，用于标识用户

**理由**：在写入时清理，保证数据一致性，避免定时任务的延迟问题。

### D5: 排行榜固定表头

**选择**：将表头绘制在固定位置（列表区域顶部），裁剪区域从表头下方开始，仅列表项参与滚动。

**实现方式**：
1. 表头 Y 坐标固定为 `listTopY`，不随 `scrollOffset` 变化
2. 裁剪区域起点从 `headerY + headerHeight` 开始
3. 列表项渲染时使用 `listStartY + i * itemHeight - scrollOffset` 计算 Y 坐标

**理由**：最直接的固定表头方案，无需额外的布局计算。

### D6: 排行榜平滑滚动

**选择**：调整滚动灵敏度，降低单次滑动的位移量，避免小幅度滑动跳页。

**实现方式**：
1. 检查 `_onScroll` 中的 `cumulativeDeltaY` 是否过大
2. 可能需要调整 `input.js` 中的速度计算或阈值
3. 确保小幅触摸移动产生小幅滚动，而非跳转

## Risks / Trade-offs

- **[风险] 回滚合并动画后重叠问题仍存在** → 用户认为原始效果更好，优先保证动画可见性
- **[风险] wx.getUserProfile 可能被用户拒绝** → 使用默认昵称"匿名用户"降级，分数不计入排行榜
- **[风险] 云端查询第 100 名存在竞态** → 当前规模下并发低，可接受

## Migration Plan

1. 云数据库 `scores` 集合需新增 `nickname` 字段，旧数据无此字段时显示"匿名玩家"
2. 回滚方案：各模块独立，可单独回退

## Open Questions

- `wx.getUserProfile` 在最新基础库中的可用性和替代方案（需测试确认）
- 平滑滚动的具体参数调整需在真机上测试
